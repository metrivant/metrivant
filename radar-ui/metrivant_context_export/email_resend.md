# Metrivant — Email (Resend)

## Overview

All outbound email routes through `lib/email.ts` — the canonical Resend email module. No SDK is used; raw `fetch()` to the Resend REST API keeps dependencies minimal.

**Design principles:**
- Graceful degradation: missing `RESEND_API_KEY` silently skips; never crashes product
- Consistent visual identity across all templates
- Idempotency where required (tracking confirmation, first-signal detection)
- Non-blocking: `void sendEmail(...)` is fire-and-forget

---

## Canonical Module (`lib/email.ts`)

### Types

```typescript
type EmailParams = {
  to:      string;
  subject: string;
  html:    string;
  from?:   string; // defaults to FROM_HELLO
};

type EmailResult = {
  ok:     boolean;
  error?: string;
};
```

### Sender Addresses

```typescript
function fromAddr(prefix: "hello" | "alerts" | "briefs"): string {
  const override = process.env.FROM_EMAIL;
  if (override) return `Metrivant <${override}>`;
  return `Metrivant <${prefix}@metrivant.com>`;
}

export const FROM_HELLO  = fromAddr("hello");   // hello@metrivant.com
export const FROM_ALERTS = fromAddr("alerts");  // alerts@metrivant.com
export const FROM_BRIEFS = fromAddr("briefs");  // briefs@metrivant.com
```

`FROM_EMAIL` env var overrides all three addresses when set (useful for single-domain Resend setups in early stage before custom subdomains are configured).

### sendEmail()

```typescript
export async function sendEmail({
  to, subject, html, from = FROM_HELLO,
}: EmailParams): Promise<EmailResult>
```

- Returns `{ ok: true }` on success
- Returns `{ ok: false, error: "..." }` on failure (key missing, HTTP error, network error)
- Logs errors via `console.error` — visible in Vercel function logs
- Never throws

## Email Templates

### Shared Layout (`emailShell()`)

All templates use a shared HTML shell:
- Background: `#f9fafb`
- Card: white, `border-radius: 12px`, `border: 1px solid #e5e7eb`
- Header: `#020802` dark green background, `#2EE6A6` accent title
- Body: light card with Inter-equivalent fonts
- CTA buttons: `#2EE6A6` background, `#020802` text, pill-shaped

This matches the Metrivant command-center color identity in email form.

### 1. Welcome Email (`buildWelcomeEmailHtml`)

**Trigger**: POST to `/api/events/signup` on new user registration
**From**: `hello@metrivant.com`
**Subject**: "Welcome to Metrivant"

Content:
- "Your competitive intelligence radar is ready."
- Brief description of the product
- CTA button: "Add your first competitor →" → `/app/onboarding`
- Footer: "Reply to this email with any questions."

### 2. Tracking Confirmation (`buildTrackingConfirmationEmailHtml`)

**Trigger**: POST to `/api/onboard-competitor` when competitor is newly added (not on duplicate submit)
**From**: `hello@metrivant.com`
**Subject**: "Your competitor radar is live"

**Idempotency**: Pre-query with `.maybeSingle()` before upsert. Email only fires if `existing === null`:
```typescript
const { data: existing } = await supabase
  .from("tracked_competitors")
  .select("id")
  .eq("org_id", org.id)
  .eq("website_url", url)
  .maybeSingle();

if (!existing && user.email) {
  void sendEmail({ ... });
}
```

Content:
- "Metrivant is now monitoring {competitor_name}."
- Monospace domain display (e.g. `lockheedmartin.com`)
- Description of what triggers an alert
- CTA: "Open radar →" → `/app`
- Footer link: "Manage alerts →" → `/app/alerts`

### 3. First Signal Email (`buildFirstSignalEmailHtml`)

**Trigger**: `/api/check-signals` — only when this org's total alert count equals the newly inserted alert count (i.e., these are the first-ever alerts for this org)

**First-signal detection logic:**
```typescript
const { count: totalAlerts } = await supabase
  .from("alerts")
  .select("*", { count: "exact", head: true })
  .eq("org_id", org.id);

const isFirstSignal = (totalAlerts ?? 0) === newAlerts.length;

if (isFirstSignal) {
  void sendEmail({ html: buildFirstSignalEmailHtml(signals, siteUrl), ... });
} else {
  void sendEmail({ html: buildAlertEmailHtml(signals, siteUrl), ... });
}
```

Content:
- "Metrivant detected its first movement signal from your competitor radar."
- Table of up to 3 signals: competitor name + signal type badge + summary
- CTA: "View on radar →" → `/app`
- Footer: "View all alerts →" → `/app/alerts`

### 4. Regular Alert Email (`buildAlertEmailHtml` from `lib/alert.ts`)

**Trigger**: `/api/check-signals` — for subsequent alerts after the first signal
**From**: `alerts@metrivant.com`
**Subject**: Varies (e.g., "3 new competitor signals detected")

Defined in `lib/alert.ts` (pre-existing module, not rewritten).

### 5. Weekly Brief Email

**Trigger**: `/api/generate-brief` — Monday 08:00 UTC
**From**: `briefs@metrivant.com`
**Subject**: "Your weekly intelligence brief — {date range}"

Generated by `lib/brief.ts` using `generateBrief()` (OpenAI GPT-4 synthesis of the week's signals).
Sent via `sendEmail({ ..., from: FROM_BRIEFS })`.

## Email Flows by Route

| Route | Email Sent | Condition |
|-------|------------|-----------|
| `/api/events/signup` | Welcome | Always (when email valid) |
| `/api/onboard-competitor` | Tracking Confirmation | Only if competitor is new |
| `/api/check-signals` | First Signal or Regular Alert | Depends on total alert count |
| `/api/generate-brief` | Weekly Intelligence Brief | Weekly Monday |

## Resend API Integration

Direct REST API (no SDK):

```typescript
const res = await fetch("https://api.resend.com/emails", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    Authorization: `Bearer ${process.env.RESEND_API_KEY}`,
  },
  body: JSON.stringify({ from, to, subject, html }),
});
```

**Environment variables:**
- `RESEND_API_KEY` — API key from Resend dashboard
- `FROM_EMAIL` — optional override for sender address (overrides all three prefix addresses)

---

## Email Address Setup Required

To send from `hello@metrivant.com`, `alerts@metrivant.com`, and `briefs@metrivant.com`:
1. Add `metrivant.com` as a verified domain in the Resend dashboard
2. Add Resend's SPF/DKIM/DMARC records to DNS

If a single sender address is preferred: set `FROM_EMAIL=alerts@metrivant.com` (or similar) to override all three defaults.

---

## Non-Blocking Pattern

All email sends use `void`:
```typescript
void sendEmail({ to: user.email, subject: "...", html: ... });
```

Email delivery failures are logged but never propagate to the user-facing response. This is intentional — email is supplementary to the product, never a blocker.
